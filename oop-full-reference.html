Object-Oriented Programming Principles

Inheritance
In short: a child class inherits the properties and behavior of its parent class.
  • A subclass reuses the fields and methods already defined on its superclass, instead of redefining them.
  • On top of what it inherits, the subclass is free to add its own extra fields and methods.
  • This is either reusing the parent's properties as-is, or extending the parent with new features.
class Vehicle {
    void start(){ System.out.println("Vehicle started"); }
}
class Car extends Vehicle {
    void honk(){ System.out.println("Car honking"); }
}
// Car has both start() and honk()
Car myCar = new Car();
myCar.start(); myCar.honk();
Abstraction
In short: separate the idea of what something does from the details of how it does it.
  • Abstraction means defining ideas in one place, so those ideas can be implemented differently by each subclass.
  • In Java, abstraction is built with abstract classes and interfaces.
  • Until Java 8, interfaces could only declare abstract methods; from Java 8 onward they can also carry concrete default and static methods.
abstract class Shape {
    abstract void draw();  // the idea, no implementation here
}
class Circle extends Shape {
    void draw(){ System.out.println("Drawing a circle"); }
}
class Rectangle extends Shape {
    void draw(){ System.out.println("Drawing a rectangle"); }
}
Polymorphism
In short: "poly" means many, "morph" means forms; one interface or method, taking many forms.
  • Any operator, method, or constructor that can be used for more than one kind of task counts as polymorphic. The classic example: + both adds numbers and concatenates strings.
  • Java has two kinds of polymorphism: static (resolved at compile time) and dynamic (resolved at runtime).
Static polymorphism: method overloading
  • Several methods share a name but differ in their parameters.
  • The compiler decides which one to call before the program even runs.
class Calculator {
    static int add(int a, int b){ return a+b; }
    static double add(double a, double b){ return a+b; }
}
Calculator.add(2, 3);      // resolved at compile time
Calculator.add(2.5, 1.5);  // resolved at compile time
Dynamic polymorphism: method overriding
  • A subclass redefines a method already declared by its superclass.
  • Also called late binding, or dynamic binding, because the version that actually runs is determined at runtime, based on the real object, not the reference type.
class Animal {
    void makeSound(){ System.out.println("Some generic sound"); }
}
class Dog extends Animal {
    @Override
    void makeSound(){ System.out.println("Bark"); }
}
Animal a = new Animal();
a.makeSound();          // prints "Some generic sound"
a = new Dog();
a.makeSound();          // prints "Bark", decided at runtime
Encapsulation
In short: bundle data and the operations on that data into a single unit.
  • Encapsulation means grouping variables (data) and the methods (operations) that act on them into one class.
  • In Java, this is done by making fields private and reaching them only through the class's own methods.
class Person {
    private String name;
    private int age;

    public String getName(){ return name; }
    public void setName(String name){ this.name = name; }

    public int getAge(){ return age; }
    public void setAge(int age){ this.age = age; }
}
Inheritance, Abstraction, Polymorphism, and Encapsulation are the four pillars behind every Java OOP design: reuse what a parent already provides, define an idea once and let each class implement it its own way, let one call take many forms, and keep an object's data and behavior bundled and guarded together.
